Лабораторная работа #7
Делаем работу в редакторе презентаций ещё удобнее.
[10] Необходимо реализовать возможность отмены совершенных действий в редакторе и отметить отмену. В редакторе должны быть 2 кнопки (Undo и Redo) для реализации этого функционала.

[10] Функционал undo/redo должен быть доступен через клавиатуру, комбинации Ctrl+Z/Ctrl-Y.
Функционал undo/redo должен работать на компьютерах под операционной системой MacOS (Command+Z/Command+Y).

При выполнении команд undo/redo должен восстанавливаться контекст изменения (должен произойти скролл к изменяемому слайду, должны выделиться изменяемые объекты и тд.)

Сторонние библиотеки для реализации функционала использовать нельзя.

Пример реализации undo/redo описан в статье: https://dzen.ru/a/Z-UZjHVcaE6yDB1F Статья про undo redo
https://redux.js.org/usage/implementing-undo-history
Важный комментарий к статье: в вашей реализации не нужно использовать полное копирование объекта (функцию structuredClone), так как у вас используются иммутабельные структуры данных.

Лабораторная работа #8
Добавляем работу с сервером в нашем редакторе презентаций. 
Для начала изучите информацию по BaaS тут:
Что такое Backend-as-a-Service (Baas)? https://procloud.ru/blog/technologies/backend-as-a-service/
и
BaaS и FaaS: как создавать приложения без настройки серверов https://sky.pro/wiki/javascript/chto-takoe-frontend-i-backend-as-a-service/

В нашей проекте мы будем использовать Appwrite. https://appwrite.io/
Appwrite - это опенсорсная BaaS система.

[10] Необходимо реализовать авторизацию и регистрацию пользователя. Для этого у вас должна быть форма логинации и регистрации пользователя. Информация о пользователе должна сохраняться в Appwrite.
При выполнении этого задания нельзя использовать сторонние библиотеки, кроме SDK Appwrite.


[20] Необходимо реализовать сохранение презентации в Appwrite. Сохранение должно происходить автоматически, по аналогии с Google презентациями. JSON-документа должен сохраняться в TableDB в Appwrite. Картинки и другие медиа файлы, если они у вас есть, должны сохраняться в Storage-е в Appwrite.
После реализации этого пункта в JSON-е должна быть ссылка на файл, а не base64 медиа файла.
Также в рамках этого задания, необходимо доработать вставку изображений в редакторе. Изображение сначала должно загружаться в Storage, после этого ссылка на это изображение должно сохраняться в state-е.
При выполнении этого задания нельзя использовать сторонние библиотеки, кроме SDK Appwrite.

[10] Валидация документа. Необходимо добавить валидацию документа сохраненного на сервере. Валидация должна происходить при загрузки документа из Appwrite. При валидации должны проверяться существуют ли все обязательные поля и тип данных этих полей. 
Для выполнения этого задания можно использовать библиотеку ajv: Ajv JSON schema validator.
